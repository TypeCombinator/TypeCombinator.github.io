在[Rank-Balanced Trees](https://sidsen.azurewebsites.net/papers/rb-trees-talg.pdf)这篇论文中，作者将AVL树和红黑树统一为了**秩**（rank）平衡树，还提出了WAVL（Weak AVL）树，相比于红黑树的节点删除最多需要3次旋转（当**缺陷子树**的兄弟节点为红色节点时），WAVL树的节点删除最多只需要2次旋转。和红黑树一样，对于节点数量为`n`的WAVL树，最大树高为`2lgn`。
<!--more-->

## Ranked-Balanced Tree

对于秩平衡树，每个节点都用秩标记，节点`x`的秩记为`r(x)`，如果将节点`x`的父节点记为`x.P`，那么父节点的秩为`r(x.P)`，节点`x`的**秩差**（rank difference）为`i = r(x.P) - r(x)`，对于秩差为`i`的节点记为`i-child`，一个节点有左右2个子节点（可以是`nil`）秩差分别为`i`和`j`，那么将该节点记为`i,j-node`（这里`i`和`j`不区分左右），特别的，叶子节点**也**可以记为`i,j-leaf`。原论文将`nil`节点的秩记为-1，叶子节点的秩记为`0`，我认为这不是一个好主意。本文将`nil`节点的秩记为0，而叶子节点的秩记为1，这样会带来一些好处，比如，对于AVL树来说，任意节点的秩等于以节点为根的子树的树高，而采用原论文的描述时，秩会比树高少1。采用**秩**和**秩差**的概念描述AVL树和红黑树如下：

- 对于AVL树，秩就是树高，所有的节点都是`1-child`或`2-child`，但是不允许`2,2-node`；

- 对于红黑树，秩就是黑高，所有的节点都是`0-child`（红色）和`1-child`（黑色），不允许父子节点都是`0-child`（红色），红黑树的各种变体只是对`0-child`的约束不一样。

如果仅仅是将AVL树和红黑树用一个框架统一起来，那么该论文将黯然失色，论文放松了AVL树的平衡条件，提出了WAVL（Weak AVL）树。对于WAVL树，所有的节点都是`1-child`或`2-child`，但是不允许叶子节点是`2,2-node`。一棵WAVL树在没有节点删除的时候就是一棵AVL树，删除WAVL树中的节点时可能会产生`2,2-node`，所以**AVL树是WAVL树的子集**。原论文已将节点插入和删除的平衡修复做了清晰的图文结合的描述，笔者选择跳过（偷懒），按照惯例，**推荐直接看[原论文](https://sidsen.azurewebsites.net/papers/rb-trees-talg.pdf)**。

## 实现方式

在写本文之前，我在网络上搜索到了[MaskRay](https://github.com/MaskRay)的名为[Weak AVL Tree](https://maskray.me/blog/2025-12-14-weak-avl-tree)的文章。有大佬的珠玉在前，本节部分内容摘录自该文章。WAVL树中只有`1-child`和`2-child`这2种节点，最低1bit就能区分，实现起来也是灵活多样，有如下几种实现方式：

1. 每个节点存储2bits的秩差，左右子节点秩差各用1bit表示，子节点对应bit为0表示秩差为1，为1表示秩差为2；

2. 每个节点存储1bit的`rank parity`，节点与其父节点的`parity`相同的是`2-child`，不同则是`1-child`；

3. 每个节点存储1bit的秩差，为0表示该节点是`1-child`，为1表示该节点是`2-child`；

4. 每个节点中直接储存绝对秩，8bits绰绰有余。

在WAVL树的实现中，FreeBSD和LLVM libc采用的是第1种方式，Fuchsia采用的是第2种方式。显然，以上4种方式并不仅仅只适用于WAVL，也适用于其他的Rank-Balanced Tree，包括AVL树和红黑树，而AVL树通常采用的是第4种方式，红黑树实现采用的是第3种方式。红黑树的根节点颜色并不重要，只是将根节点染黑在使用第3种方式实现时可以省去一些处理或判断，比如父节点是红色时，那么祖父节点必然存在。如果将红黑树改用第1种方式实现，则更能体现出根节点颜色的不重要性。[Succinct Encodings of Binary Trees with Application to AVL Trees](https://arxiv.org/pdf/2311.15511)这篇论文证明了AVL树的每个节点只需要约0.938比特的信息。

## 性能差异

在之前的文章中，笔者提到FreeBSD的WAVL树实现在节点插入上表现出的优异性能是因为具体实现造成的。在没有节点删除时，WAVL树就是AVL树，所以单看WAVL树的节点插入性能应当与AVL树完全一致。在相同的实现方式的前提下，AVL树的节点插入性能通常比红黑树（限定2-3红黑树或2-3-4红黑树）要差一些，尽管AVL树的平均树高更低。FreeBSD的WAVL树的节点链接指针采用了数组，部分代码如下：

```c++
#define RB_ENTRY(type)                     \
    struct {                               \
        struct type *rbe_link[3];          \
    }

#define _RB_LINK(elm, dir, field)	(elm)->field.rbe_link[dir]
#define _RB_UP(elm, field)		_RB_LINK(elm, 0, field)
#define _RB_L				((__uintptr_t)1)
#define _RB_R				((__uintptr_t)2)
#define _RB_LR				((__uintptr_t)3)
```

这种实现方式不仅能少写一半平衡修复代码，产生更少的指令，更为重要的是还能减少分支。笔者在实现WAVL树时采用了每个节点用2bits储存左右子节点秩差的方案，这与FreeBSD的WAVL树的实现方式一样，但是笔者没有选择将链接指针用数组聚合到一起，而是让链接指针以及秩信息各自独立，原因如下：

- 链接指针不应当用结构体或数组聚合到一起，几比特的秩信息会导致内存填充，且不方用户调整内存布局，更会削弱组合能力，如果将链接指针拆分并复用，节点在不同类型的侵入式容器中迁移会变得非常容易；
- 对于C\+\+来说，是不需要将平衡树的秩信息保存在指针未用比特的这种奇技淫巧的，因为秩信息成员可以独立出来，通常节点都会有一些未用到的比特，可以让用户决定用哪个成员的哪些比特。

## 漏网之鱼

**WBT（Weight-Balanced Tree）也是Rank-Balanced Tree**，所谓**秩**可以看作对树高的估计，我们可以将WBT中节点的Weight取以2为底的对数作为该节点的秩。在WBT中，对于节点`v`，记权重为`v.S`，左右节点分别表示为`v.L`和`v.R`，为了便于描述，和之前的文章不同，这里的叶子节点权重记为2，如果WBT平衡，则所有子树都满足以下条件：

- `v.L.S * ∆ ≥ v.R.S`
- `v.R.S * ∆ ≥ v.L.S`

而根节点Weight可以由左右子节点的Weight相加获得，代入`v.S = v.L.S + v.R.S`，再取一下对数可转换为如下不等式：

- `lg(∆ + 1) ≥ lg(v.S) - lg(v.L.S)`
- `lg(∆ + 1) ≥ lg(v.S) - lg(v.R.S)`

我们将`lg(v.S)`记为该节点的秩，即`rank(v)=lg(v.S)`，最终整理如下：

- `lg(∆ + 1) ≥ rank(v) - min(rank(v.L), rank(v.R))`

WBT的平衡条件可以描述为**最大秩差**小于等于`lg(∆ + 1)`，而`∆ = 3`时对应最大秩差为2。与其他Rank-Balanced Tree不同之处在于：WBT中的节点的秩或者秩差可能是**小数**。

## 自顶向下

WBT性能不佳的一个关键原因在于，如果采用自底向上的平衡维护策略，在节点增删时，Weight总是要从树底一直更新到树根。当采用自顶向下的平衡维护策略时，只需要单趟寻路，会显著改善WBT的性能。对于红黑树、AVL树或WAVL树，只会在少数情况下需要自底向上地维护到根节点，因此将这些平衡树改用自顶向下维护策略所得到的收益并不多，而且自顶向下的维护还有局限性，比如不太适用于不允许key重复的场景。

所有的Rank-Balanced Tree都支持自顶向下维护，但**自顶向下维护策略并不总是一个好的选择**。对于AVL树或WAVL树，如果采用自顶向下的维护策略，元素插入需要5个节点的look-ahead，而需要的look-ahead越多实现越复杂，所以自顶向下AVL树或WAVL树的实践意义可能并不大。对于并发场景，如果要实现无锁平衡二叉树，自顶向下红黑树是一个值得考虑的选择。而在需要查询节点排名或第k大节点的场景中，则更适合选用WBT。

## 看山还是山，看水还是水

当Rank-Balanced Tree不平衡的时候，会用到两种手段修复平衡：

- 一为配平，具体有升秩（Promotion）或降秩（Demotion），这对应了红黑树中的染色，配平用左右子树计算得到的秩，并使秩差满足平衡约束；
- 二为旋转，秩越大的子树越高，所以兄弟节点中秩差越小的子树越高，而所谓旋转就是将更高的子树向更低的兄弟子树调整，选取更接近二分的节点作为树根，使得树更平衡。
